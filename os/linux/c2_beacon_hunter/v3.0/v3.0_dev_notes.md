### Overview

- The ML engine has been evolved from process-level to **network-level (subnet/CIDR)** behaviour.
- Baselines and clustering now operate on CIDR blocks (/24 for IPv4, /64 for IPv6) instead of individual processes.
- This dramatically improves detection of subnet-wide C2 campaigns while reducing alert fatigue on single-host noise.
- Support for environments where host eBPF is impossible (cloud VMs, serverless, restricted containers).
- The system now ingests native cloud flow logs (AWS VPC, Azure NSG, GCP VPC) via a generic JSON-lines adapter.
- Dual-backend support (SQLite or Postgres) with one-time migration script.
- Interactive SOC console with full event drill-down and timeline heatmaps.
- Distributed architecture allowing lightweight endpoints (Agents) to securely forward eBPF telemetry to a Central NDR Server via pinned HTTPS.
- Transformed the platform into a SOAR-lite ecosystem (block + isolate + notify).
- Implemented a "Brain vs. Muscle" architecture to handle distributed containment across multiple endpoints.

### Completed Stories

**Story 1.1: TC/XDP program in promiscuous mode**
- Full XDP parser (`c2_promisc.bpf.c`) with robust bounds checking
- IPv4 + IPv6 support (5-tuple extraction)
- In-kernel ringbuf emission for flows
- Safe attach/detach logic (prevents -EBUSY)

**Story 1.2: Robust Ethernet/IP/TCP/UDP header parsing**
- Complete header parsing with early-exit safety
- Payload sampling (first 64 bytes) for future entropy
- Zero-copy, wire-speed operation

**Story 1.3: In-kernel flow state tracking (5-tuple maps)**
- Ringbuf-based flow events (no per-packet spam)
- Dual-mode loader selection (`c2_promisc_loader` vs original)

**Story 2.1:** BPF Hash Maps for interval, entropy, CV, and packet-size tracking

**Story 2.2:** Kernel-side aggregation before ringbuf submission
→ Dramatically reduces ringbuf pressure and user-space CPU load. Events are now aggregated every 8 packets (or on new flow).

**Story 3.1:** CIDR-based baselines (10.0.5.0/24, etc.) — implemented in `baseline_learner.py`

**Story 3.2:** 3D+ clustering with flow metadata (interval + entropy + subnet diversity score) — implemented in `BeaconML.py`

**Story 4.1:** Adapters for AWS VPC Flow Logs, Azure NSG, GCP (auto-detect + JSON-lines)

**Story 4.2:** Pure flow-log mode with reduced entropy features (no payload, no PID/process tree)

**Story 5.1:** Dual-backend support (SQLite for single-host, Postgres for enterprise)

**Story 5.2:** SQLAlchemy + asyncpg for high-performance writes

**Story 5.3:** Migration script (`sqlite_to_postgres.py`)

**Story 5.4:** Partitioning by day + process_name for fast queries

**Story 5.5:** Connection pooling and read replicas for SOC dashboard scale

**Story 5.6:** Baseline learner updated to use Postgres for cross-host UEBA

**Story 6.1:** Interactive SOC Dashboard UI
- Implemented `index.html` with a cyberpunk aesthetic (Tailwind CSS) and dynamic DOM updates.
- Real-time WebSocket integration for live anomaly pushing directly to the browser.

**Story 6.2:** Advanced Drill-Down & Visualizations
- Clickable event rows triggering detailed modals containing Process Trees, ML features, and MITRE mapping.
- Integrated Chart.js for an interactive 7-day timeline heatmap with zoom/pan capabilities.
- Dynamic PCAP generation and single-event JSON exports directly from the UI.

**Story 6.3:** FastAPI Backend Enhancement
- Developed `api_server.py` to handle advanced queries, flow context lookups, and dynamic PCAP byte-stream generation.

**Story 7.1:** Lightweight Agent Mode Architecture
- Created `agent_runner.py` to bypass local databases and act purely as a secure forwarder.
- Updated `libbpf_collector.py` to dynamically route based on environment (local DB vs HTTPS POST).

**Story 7.2:** Secure Central Ingestion
- Implemented `generate_certs.sh` for easy self-signed CA and server certificate creation.
- Enforced strict SHA256 CA Certificate Pinning via environment variables (`CA_CERT_HASH`) to prevent MITM attacks on agent telemetry.

**Story 8.1: Centralized SOAR Dispatcher**
- Refactored `c2_defend.py` on the Central server to act as a command dispatcher rather than a local execution script.
- The dispatcher monitors the global `anomalies.jsonl` stream and identifies the originating `agent_id` for every high-score detection.
- Implemented an automated "Decision Engine" that triggers containment orders only when the anomaly score exceeds a critical threshold (Default: 90).

**Story 8.2: Secure Command & Control (C2) Channel**
- Enhanced `api_server.py` with a dedicated WebSocket registry (`agent_connections`) to maintain persistent, encrypted links to all active agents.
- Added an internal SOAR API endpoint (`/api/v1/soar/contain`) that allows the dispatcher to push JSON-formatted "Kill/Block" orders to specific endpoints.

**Story 8.3: Remote Execution Muscle (Agent Action)**
- Upgraded `agent_runner.py` with an asynchronous command listener that handles sub-millisecond containment orders.
- The agent now executes surgical local mitigations upon receipt of a central order, including:
    - **Process Eradication:** Instant `SIGKILL` of malicious PIDs.
    - **Network Isolation:** Automated XDP map updates and OS firewall (iptables/ufw/firewalld) drops.

- CIDR-based baselines (`10.0.5.0/24`, `2001:db8::/64`) instead of per-process
- 3D+ clustering in BeaconML (time series + entropy + subnet diversity)
- Subnet diversity scoring for campaign detection
- Health check & Docker awareness of new mode
- Zero performance impact when running in host mode
- Full migration script (sqlite_to_postgres.py)
- Connection pooling + partitioning in Postgres mode
- Real-time WebSockets for instant anomaly alerts in the UI.
- Comprehensive Drill-Down Modals with process trees, ML feature breakdowns, and mitigation status.
- Interactive 7-day Timeline Heatmaps utilizing Chart.js.
- On-the-fly JSON and PCAP export generation for DFIR analysts.
- Lightweight Agent Forwarding with strict SHA256 CA Certificate Pinning.
- Centralized API Ingestion supporting large-scale, multi-endpoint environments.
- Automated "Block + Isolate + Notify" playbooks.
