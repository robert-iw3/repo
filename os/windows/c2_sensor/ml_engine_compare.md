### The Retrospective (Python V6 vs. Rust V1)

| Capability | Python V6 (`BeaconML.py`) | Rust V1 (`C2Sensor_ML.dll`) | Impact / Assessment |
| :--- | :--- | :--- | :--- |
| **Execution Model** | Standalone Daemon (`python.exe`). Relied on STDIN/STDOUT pipes fed by PowerShell. | Native `cdylib` mapped into C# ETW memory via `[DllImport]`. | **Massive Upgrade.** IPC pipes are notoriously brittle on Windows and inherently bottlenecked by IO serialization limits. FFI passes pointers natively, achieving near zero-latency telemetry evaluation. |
| **Dependency Footprint** | Required deploying Python 3.11, `numpy`, `scikit-learn`, `scipy` (~200MB footprint). | Zero external dependencies. Compiled into a single statically linked `.dll` (~5-10MB). | **Massive Upgrade.** Eliminates endpoint friction, supply chain vulnerabilities, and the need for messy environment variable bootstrapping (`Setup-C2HunterEnvironment.ps1`). |
| **Concurrency / Threading** | Restricted heavily by the Python GIL (Global Interpreter Lock). Required environment hacks (`OMP_NUM_THREADS=1`) to prevent C-backend deadlocks. | Fully thread-safe. Rust's ownership model and `Mutex<BehavioralEngine>` guarantee safe concurrent telemetry ingestion from the ETW firehose. | **Upgrade.** Rust will easily handle 10x the event volume of the Python script without CPU exhaustion. |
| **State Persistence** | Hybrid RAM + JSON disk flushes (`C:\Temp\C2_StateDB\*.json`). Vulnerable to disk I/O locks and file fragmentation. | SQLite database operating in Write-Ahead Logging (WAL) mode (`C:\ProgramData\C2Sensor\Data\C2Sensor_State.db`). | **Upgrade.** SQLite WAL is ACID compliant, preventing corruption during hard crashes while maintaining the temporal "Low and Slow" beacon memory across reboots. |
| **Machine Learning Math** | `scikit-learn` K-Means and DBSCAN. Heavily optimized C backends. | `linfa-clustering` + Native Euclidean Silhouette scoring. | **Parity.** The math is identical. `scikit-learn` might have a slight edge in vectorization for matrices of millions of rows, but for our specific use case (evaluating arrays of ~2000 intervals), Rust will execute the K-Means iterations in microseconds. |
| **DGA & Heuristics** | Python regex, string entropy, and list comprehensions. | Native Rust regex and HashMaps. | **Parity/Slight Upgrade.** Rust string parsing and iteration are natively faster than Python, guaranteeing the AppGuard and ETW pipelines are never blocked waiting for the ML evaluation. |
